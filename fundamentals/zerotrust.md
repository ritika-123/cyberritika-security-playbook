# 🔐 Zero Trust Architecture

## 📌 What is Zero Trust?

Zero Trust is a security model based on the principle:

👉 **“Never trust, always verify”**

It means no user, device, or system is trusted by default—even if they are inside the organization’s network.

Zero Trust का मतलब है कि system में कोई भी automatically trusted नहीं होता, हर access को verify करना पड़ता है।

---

## 🧠 Understanding Through a Scenario

Momo is working remotely and tries to access company data.

Even after logging in:

- System checks device security  
- Verifies location  
- Validates permissions  

If something looks suspicious, access is denied.

👉 Even though Momo is a valid user, trust is never assumed.

---

## 🔑 Key Concept

Zero Trust focuses on:

- Continuous authentication  
- Strict access control  
- Verification at every step  

👉 Trust is not permanent—it must be earned every time.

---

## 🔄 Core Principles

### 1. Verify Explicitly
- Always authenticate and authorize  
- Use multiple factors (MFA)

### 2. Least Privilege Access
- Give minimum permissions  

### 3. Assume Breach
- Design system as if attacker is already inside  

हर request को validate किया जाता है, चाहे user internal ही क्यों न हो।
